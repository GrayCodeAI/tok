**Date:** April 7, 2026  
**Status:** ✅ Research Complete

## 🎯 The Bottom Line

**TokMan is in a crowded, competitive market.** We face 4 major competitors and 11+ minor ones.

---

## 🔴 Top Threats

### 1. RTK (Rust Token Killer)
- **GitHub:** https://github.com/rtk-ai/rtk
- **Language:** Rust
- **Threat:** 🔴 **MAJOR** - Near-identical positioning
- **Strengths:** <10ms overhead, 100+ commands, Homebrew, 6 languages, active Discord
- **Weaknesses:** Fewer layers (~15 vs 31), no quality metrics

### 2. OMNI (Semantic Signal Engine)
- **GitHub:** https://github.com/fajarhide/omni
- **Language:** Rust
- **Threat:** 🔴 **MAJOR** - Most innovative
- **Strengths:** RewindStore (zero-loss), learning mode, MCP native, session recovery
- **Weaknesses:** Semantic only (vs 31 layers), no quality metrics

### 3. Snip
- **GitHub:** https://github.com/edouard-claude/snip
- **Language:** Go
- **Threat:** 🟡 **MODERATE** - Simplicity appeal
- **Strengths:** YAML pipelines (easy extensibility), Homebrew, OpenCode plugin
- **Weaknesses:** Only ~20 filters vs 97+, no quality metrics

### 4. Token Optimizer MCP
- **GitHub:** https://github.com/modelcontextprotocol/token-optimizer-mcp
- **Language:** TypeScript
- **Threat:** 🟡 **MODERATE** - Different approach
- **Strengths:** 65 MCP tools, diff-based updates, API/DB optimization
- **Weaknesses:** Not standalone (needs Node), MCP-only
