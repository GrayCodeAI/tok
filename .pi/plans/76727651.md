# Comprehensive Industry Standards Audit

## Issues Found

### 1. Error Wrapping Inconsistencies
- `fmt.Errorf` with `%v` instead of `%w` in some places breaks error chain unwrapping
- Need to check all error wrapping patterns

### 2. File Permission Security
- Multiple `os.MkdirAll` calls using `0755` (world-readable/writable) for config/data dirs
- Should use `0700` for user-private directories (consistent with tracker.go which uses `0700`)
- Affects ~40+ locations across the codebase

### 3. Unchecked Error Returns
- `os.MkdirAll` without error checking in several places (pathshim, toml/registry, filter/rtk_stages, memory)
- `os.WriteFile` without error checking in pathshim

### 4. Environment Variable Mutation
- `os.Setenv("SKIP_ENV_VALIDATION", "1")` in root.go modifies global process state - not thread-safe

### 5. Pathshim Security
- pathshim/shim.go creates executable scripts (`os.WriteFile` with 0755) in /tmp without sanitizing command name - potential injection

### 6. Global State in shared/
- Global mutable state pattern (documented as temporary but persists)

### 7. go fmt violations
- 8 files needed auto-formatting

### 8. math/rand vs crypto/rand
- Non-cryptographic rand used in several packages - acceptable for experiments/chaos but should use `rand.New(rand.NewSource(...))` for Go 1.20+ behavior

### 9. Large file issues
- tracker.go is massive (~1200+ lines) - violates single responsibility
- config.go is very large - could be split

### 10. Missing context propagation
- Several DB methods don't accept context.Background()
- Run queries without cancellation support

### 11. SQL injection patterns
- Dynamic SQL query building with string concatenation in several tracker methods
- GLOB pattern injection possible through user input

### 12. Resource leaks
- deferred Close() after Open without nil check in some places
