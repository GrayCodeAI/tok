Add SQLite-based persistent cache for filtered command outputs. Cache results by command fingerprint for instant retrieval on repeated commands.

## Goals
- Cache filtered outputs to SQLite
- Key by command + args + working_dir hash
- 56s → 1s speedup on cache hits
- Invalidate on file changes (git integration)
- Integrate with MCP server and CLI

## Implementation
- internal/cache/query_cache.go - Core caching logic
- Key: SHA256(command + args + cwd + file_hashes)
- Value: filtered_output, original_tokens, filtered_tokens, timestamp
- Git watcher for automatic invalidation
- Cache hit/miss metrics
